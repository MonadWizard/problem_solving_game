# Architecture

## Module map

```
public/data/*.json          content: journeys, items, story (see curriculum skill)
src/lib/types.ts            shared types: Problem, Journey, Solve, GameEvent, ProgressState, LocalState, QueueOp
src/lib/xp.ts               level/bounty math, totalXp derivation
src/lib/streak.ts           streak from event log (+freezes)
src/lib/quiz.ts             pattern-gate quiz generator (seeded)
src/lib/ghosts.ts           ghost-ship spaced repetition (7/21/60d)
src/lib/unlocks.ts          island/journey unlock + progress + ship tier
src/lib/poster.ts           SVG avatar + canvas bounty-poster renderer
src/lib/verify.ts           optional LeetCode verification (worker-flagged)
src/data/curriculum.ts      fetch + index journeys/items/story (BASE_URL-aware)
src/store/localStore.ts     versioned localStorage persistence (tga:v1)
src/store/gameStore.ts      zustand store: ALL mutations (local-first, enqueue cloud op)
src/sync/merge.ts           pure merge (idempotent+commutative) — property tested
src/sync/queue.ts           offline retry queue (persisted in LocalState)
src/sync/cloud.ts           CloudAdapter interface + SupabaseCloud impl
src/sync/supabaseClient.ts  lazy client; null ⇒ guest-only mode
src/sync/engine.ts          onLogin merge, reconcileOnOpen, online/auth listeners
src/auth/AuthProvider.tsx   session context, OAuth + email/password
src/screens/*               MapScreen, IslandScreen, ProfileScreen, LeaderboardScreen, AboutScreen
src/components/*            SeaChart, IslandNode, ShipSprite, GhostShip, ProblemRow, BossCard,
                            QuizModal, AttemptTimer, ChestToast, BountyPoster, ItemShelf, AuthMenu, SavePrompt
supabase/schema.sql         tables, RLS, triggers, leaderboard fns; seed.sql generated by scripts/gen-seed.mjs
worker/                     optional Cloudflare Worker (LeetCode verification proxy)
```

## State & data flow

UI → `gameStore` action → (1) mutate LocalState + `saveLocal` (instant) → (2) push QueueOp → background `flushQueue` to Supabase. Reads: selectors derive everything (XP, level, streak, unlocks, ghosts) from `ProgressState` + curriculum. Login/app-open runs merge (see supabase-sync skill). Routing: HashRouter — `/`, `/island/:journeyId/:islandId`, `/profile`, `/leaderboard`, `/about`.

## Testing

Vitest (jsdom). Pure-logic suites: curriculum validation, xp, streak, quiz, ghosts, unlocks, localStore, merge (fast-check properties), queue, sync integration (FakeCloud, device A/B resume equality).
